Aide sur Git : généralités

Définir son nom et email :
git config --global user.name "Votre nom ou pseudo"
git config --global user.email "votre@email.com"

Pour activer un dossier comme repository Git, se placer dans ce dossier puis :
git init

Pour ajouter à l'index Git un fichier nouvellement créé :
git add checklist-vacances.md

Pour ajouter tous les fichiers du répertoire courant :
git add .

Après une modification du projet :
git commit -m "message pour commenter le commit"

Pour mettre à jour tous les fichers déjà présents dans l'index :
git commit -a -m "message pour commenter le commit"

Pour afficher l'historique des commit :
git log

Pour se positionner sur un commit :
git checkout SHADuCommit

Pour se positionner sur le dernier commit :
git checkout master

Pour déterminer l'état des fichiers de l'application :
git status

Aide sur Git : les branches

Pour afficher les branches :
git branch

Pour créer une nouvelle branche :
git branch nouvelle-branche

Pour se positionner sur une branche :
git checkout ma-branch

Les deux dernières opérations en une seule :
git checkout -b nouvelle-branche

Ajouter dans une branche A les mises à jour effectuées dans une branche B :
git checkout branche-A puis git merge branche-B

Règlement d'un conflit :
- Modifier la branche confictuelle
- Git commit
- :x pour sauvegarder le message tel quel

Pour supprimer une branche :
git branch -d ma-branche

Aide sur Git : commandes diverses

Qui a modifier un fichier du projet :
git blame nomdufichier.extension puis git show débutdusha

Pour ignorer certains fichiers dans Git, utiliser le fichier .gitignore

Dans ce fichier, lister les fichiers de configuration, ou fichies contenant des mots de passe :
monrepertoire/motdepasses.txt
config/application.yml

Les deux dernières opérations en une seule :
git checkout -b nouvelle-branche

Ajouter dans une branche A les mises à jour effectuées dans une branche B :
git checkout branche-A puis git merge branche-B

Règlement d'un conflit :
- Modifier la branche confictuelle
- Git commit
- :x pour sauvegarder le message tel quel

Pour éviter de faire un commit superflu, mettre de coté les modifications par :
git stach puis les reprendre par git stash pop

Aide sur GitHub

Pour récupérer un repository sur sa machine, ce positionner sur le répertoire de réception et :
git clone lienhttpsfournipargithub

Pour créer un repository sur gitHub, cliquer sur le bouton "Create new" ou "+" du site
Pour synchroniser le repository local et celui de GitHub, se placer sur le répertoire local et :
git push origin master

Inversement, pour récupérer les modifications effectuées sur GitHub sur sa machine locale :
git pull origin master
